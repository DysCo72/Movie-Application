REST API Document


The REST API allows users to obtain information about the movies or categories for oscars Awards.

This pages list instructions for the use of the API.

Find Nomination by ID

An Oscars nomination can be located in the h2 database.

Find Nomination by ID would always return only one JSON object.

Input:

http://localhost:8080/awardmovies/id/[id]

[id]: Valid ID will be integer with a value between 1 and 10394.

Output/Return:

A single JSON object with this format:

Example: {"id":1,"title":"The Noose","category":"ACTOR","yearReleased":1927,"yearNominated":1928,"awardee":"Richard Barthelmess","awardStatus":false,"link":"https://www.themoviedb.org/movie/113167/watch","imageLink":"https://image.tmdb.org/t/p/w500/hheIsv2RAp8htWZspOPPEcEdlTL.jpg","plot":"In this suspenseful silent crime drama, a hijacker proves his loyalty to his mother by killing his biological father, a blackmailing gangster who has been threatening to destroy the mother's happy marriage to the governor.","tmdbId":113167}

id:(data type: integer) Unique id for each nominated movie

title:(data type: string) Movie's title

category: (data type: string) The Oscars nomination category

yearReleased: (data type: integer) The release year of the movie

yearNominated: (data type: integer) The nominated year of the movie

awardee: (data type: string) The awardee of the nominated movie

awardStatus: (data type: boolean) Set to false if the nominated movie did not get the award. Set to true if the ominated movie get the award.

link: (data type: string) Providing a link to an external site for the movie from an outside source TMDB.

imageLink: (data type: string) Providing a image link from the TMDB API of each movie

plot: (data type: string) A brief introduction of the movie, obtained from the TMDB API

tmdbId: (data type: integer) Unique id for the movie from the TMDB API. It is not the same as the id.

Find Nomination by year

Find all the nomination by year no matter of the categories for rewards.

The set of data contain about 100+ data, therefore, request may take a longer response time

Input:

http://localhost:8080/awardmovies/year/[year]

[year]: Valid year will be integer with a value between 1928 and 2020.

Output/Return:

A set of JSON objects of nominated movies within input year.

The format of a JSON object follow the format of nomination by ID

Find Nomination by Categories for rewards

Find Nomination by Categories will result a large set of data, and it will result a longer response time

So, it is limited to use with year.

http://localhost:8080/awardmovies/[categories]/[year]

[year]: Valid year will be integer with a value between 1928 and 2020.

[categories]:

  • animated-feature
  • actor
  • actress
  • art-direction
  • costume-design
  • dance-direction
  • cinematography
  • writing
  • directing
  • engineering-effects
  • film-editing
  • music

Output/Return:

A set of JSON objects of nominated movies within input categories and year.

The format of a JSON object follow the format of nomination by ID

Search

Capitalization doesn't matter

Input:

http://localhost:8080/awardmovies/search-movie?Title=[movie name]

[movie name]: Movie name could be genral or specific

The more genral input, the longer response time, and it might included more objects

For example: Input "the last command" will return only one JSON object

Input "last" will return about 50 JSON object with longer response time.
